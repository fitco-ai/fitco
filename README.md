# Fitco

Fitco 프로젝트는 여러 개의 프로그램 코드가 모여 하나의 큰 시스템을 이루는 구조(모노레포)로 만들어진 웹 서비스입니다.

## 프로젝트 소개 (About)

Fitco는 **Cafe24 기반 쇼핑몰을 위한 B2B(기업 간 거래) 소프트웨어 솔루션**입니다. 
주요 타겟은 패션 쇼핑몰들이며, 핵심 기능은 다음과 같습니다:

- **데이터 자동 수집 및 인공지능 분석**: 쇼핑몰에 등록된 세트 상품, 사이즈 정보, 이미지 등의 데이터를 자동으로 수집하고, 인공지능(AI)을 통해 분석합니다.
- **Cafe24 연동**: Cafe24 쇼핑몰 시스템과 연결하여 상품 정보, 옵션, 장바구니, 주문, 고객 데이터를 주고받습니다.
- **관리자용 웹 페이지 (어드민)**: 시스템에 등록된 쇼핑몰들을 관리하고, 자동 수집된 데이터를 직접 확인할 수 있는 중앙 관리 시스템 화면을 제공합니다.
- **쇼핑몰 화면용 도구 (위젯)**: 쇼핑몰 웹사이트에 직접 들어가서 고객들이 옷 사이즈를 추천받거나, 원하는 상품을 추천받을 수 있도록 도와주는 화면 도구입니다.
- **알림 및 메시지 관리**: 고객들에게 알림톡, 문자 메시지(SMS), 이메일 등의 메시지를 자동으로 보낼 수 있는 기능입니다.

## 프로젝트 구성 요소 (Architecture)

프로젝트는 크게 두 가지 공간(`apps`와 `packages`)으로 나누어 관리됩니다.

- **관리자 백오피스 (apps/admin)**: 관리자가 접속해서 사용하는 웹 화면입니다.
- **데이터베이스 모듈 (packages/database)**: 데이터를 저장하고 불러오는 방식에 대한 규칙들을 모아둔 곳입니다.
- **공통 디자인 모듈 (packages/design-system)**: 여러 화면에서 공통으로 재사용하는 버튼이나 입력창 같은 화면 디자인 부품들입니다.
- **이메일 모듈 (packages/email)**: 이메일을 전송하는 기능들이 모여있습니다.
- **데이터 분석 모듈 (packages/analytics)**: 웹사이트 방문자 수 같은 통계와 데이터를 추적하는 기능들입니다.
- **기타 설정 파일들**: 프로젝트가 잘 돌아가기 위한 환경 설정들입니다.

## 처음 시작하기 (Getting Started)

프로젝트를 내 컴퓨터에서 실행해보기 위해 필요한 기본적인 준비 과정입니다.

### 1. 환경 변수 설정
프로젝트 폴더 최상단에 있는 `.env.example` 파일의 내용을 복사해서 새롭게 `.env` 라는 이름의 파일을 만들고, 거기에 필요한 비밀번호나 기본 설정값들을 채워줍니다.
```bash
cp .env.example .env
```

### 2. 필수 프로그램 설치
이 프로젝트를 실행하는 데 필요한 외부 프로그램 파일들을 설치하는 명령어입니다.
```bash
pnpm install
```

### 3. 데이터베이스(저장소) 설정 적용
미리 정의해둔 데이터 설계도를 실제 데이터베이스 저장소에 적용하는 단계입니다. (사전에 1번에서 만든 `.env` 파일 안에 데이터베이스 접속 정보가 잘 들어가 있어야 합니다.)
```bash
pnpm migrate
```

### 4. 내 컴퓨터에서 실행하기
준비가 다 끝났다면, 아래 명령어로 내 컴퓨터에서 프로그램을 실행해볼 수 있습니다.
```bash
# 전체 서비스를 한 번에 실행하기
pnpm dev

# 관리자(어드민) 서버만 따로 실행하고 싶을 때
pnpm admin
```

## 주요 명령어 모음

개발이나 관리를 하면서 주로 쓰게 되는 명령어들입니다.
- `pnpm build`: 실제 서비스에 배포하기 위해 최종 결과물을 만들어내는 명령어
- `pnpm lint` & `pnpm format`: 코드가 규칙에 맞게 잘 작성되었는지 검사하고 정리해주는 명령어
- `pnpm test`: 프로그램의 주요 기능이 잘 작동하는지 자동으로 검사하는 명령어
- `pnpm analyze`: 완성된 프로그램 파일의 용량이 얼마나 되는지 분석하는 명령어
- `pnpm clean`: 설치했던 외부 프로그램 파일들을 깨끗하게 지우고 초기화하는 명령어

## 배포 가이드 (Deployment)

실제 서비스 서버에 프로그램을 올리는 과정입니다.

### 기본적인 단계

```bash
# 1. 서비스 운영을 위한 최종 파일 만들기
pnpm build --filter admin

# 2. 만들어진 최종 파일(.next/standalone)을 사용해 서버에서 실행하기
node apps/admin/.next/standalone/server.js
```

### AWS EC2 배포 방법

기본적으로 macOS 기반에서 작성하였습니다. 

#### 1. Terminal 실행 

Terminal 앱을 실행합니다.

#### 2. 서버 접속 

```bash
ssh -i [인증키 파일 경로] ubuntu@[EC2 서버 IP 주소]
```

인증키 파일 관련하여 권한 오류가 있는경우 400 권한을 주어줍니다.
```bash
chmod 400 [인증키 파일 경로]
```

#### 3. 서비스 배포 및 실행 

서버 접속이 완료되었다면, 다음과 같이 서비스 빌드 및 실행을 합니다. 

```bash
# 서비스 최종 파일 만들기(빌드)
pnpm build --filter admin

# pm2를 이용해 만들어진 서버를 실행하기
pm2 start apps/admin/.next/standalone/server.js --name "admin"

# 또는,
pm2 restart --name "admin"
```
